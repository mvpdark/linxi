## 三、各页面UI问题详细分析

### 1. 首页 (Home)

**页面内容**: 标题"灵犀 - 设计师的AI助手" + 三个功能卡片(灵感/改图/全景) + 最近记录列表

#### 发现的问题

| 优先级 | 问题类型 | 具体描述 | 建议 |
|:---|:---|:---|:---|
| P1 | 颜色不一致 | 三个功能按钮使用了不同色系(绿/棕/粉)，增加了认知负担 | 统一使用品牌主色，或降低辅助色饱和度差异 |
| P2 | 间距异常 | 卡片间距(~12px)与卡片到区域标题间距(~32px)比例不精确，缺乏8px网格系统 | 建立8px基网格：卡片间距16px，区域间距32px/40px |
| P2 | 间距异常 | 最近记录列表项之间缺乏明确分隔，仅靠阴影/背景区分，边界感弱 | 增加1px分割线或增大项间距至8px |
| P3 | 对齐问题 | 左对齐层级过多：图标左边缘、标题左边缘、记录文字左边缘形成三条不同轴线 | 统一卡片内部左内边距 |
| P3 | 布局偏移 | 页面底部大量留白，内容集中在页面上半部分 | 确认设计意图，或增加快捷入口/推荐内容填充 |

---

### 2. 聊天页 (Chat)

**页面内容**: 左侧窄边栏(图标导航) + 顶部标题栏 + AI欢迎消息 + 底部输入框

#### 发现的问题

| 优先级 | 问题类型 | 具体描述 | 建议 |
|:---|:---|:---|:---|
| P0 | 重复内容 | 侧边栏"会话列表"内容出现重复：body文本显示"新对话 会话列表 新对话 新对话 新对话 清除记忆"重复了两次，疑似渲染Bug | 检查Vue/React组件是否重复挂载，排查v-for/key或条件渲染逻辑 |
| P0 | 图标重复 | 侧边栏出现三个加号(+)图标(顶部、输入框左侧、左下角)，功能混淆 | 差异化图标：侧边栏用"新建对话"，输入区用"附件/添加"，明确左下角用途或移除 |
| P1 | 间距异常 | 头像与气泡间距偏大(~12-16px)，关联性弱 | 优化至8-10px增强关联性 |
| P1 | 布局问题 | 输入区与聊天区仅靠背景色区分，无明确分隔线 | 增加顶部阴影或色块分隔 |
| P2 | 间距系统 | 缺乏统一的8px/4px基线网格，各区域间距独立设定 | 建立统一间距系统 |
| P2 | 按钮状态 | 功能图标缺少hover状态指示 | 增加hover/active状态反馈 |

---

### 3. 改图页 (Image-Edit)

**页面内容**: 顶部导航栏(返回+标题"改图") + 居中虚线上传区域

#### 发现的问题

| 优先级 | 问题类型 | 具体描述 | 建议 |
|:---|:---|:---|:---|
| P0 | 布局偏移 | 底部存在巨大空白区域(约占页面50%以上)，内容过度空旷 | 添加"最近编辑"记录/热门模板/功能说明卡片；或将上传区垂直居中 |
| P1 | 视觉层级 | 虚线边框对比度不足，在浅色背景下不够醒目 | 增加hover状态(边框色变深)或轻微阴影 |
| P1 | 按钮缺失 | 整个上传区作为隐形点击热区，无明确实体按钮 | 增加"选择图片"实体按钮，降低认知成本 |
| P2 | 颜色一致性 | 绿色图标与"改图"功能关联性弱，页面缺乏统一品牌识别色 | 建立功能色彩体系 |
| P2 | 文字提示 | "自动压缩到1024px"可能引起用户画质担忧 | 增加提示图标或展开说明 |

---

### 4. 全景页 (Panorama)

**页面内容**: 顶部导航栏 + 三个模式卡片(上传成品/AI生成/拍照模式) + 历史全景列表(4条记录)

#### 发现的问题

| 优先级 | 问题类型 | 具体描述 | 建议 |
|:---|:---|:---|:---|
| P0 | 文字截断 | 历史列表文件名(如`panorama_1784274115_a7bf28.png`)无截断处理，更长文件名会撑破布局 | 添加CSS: `overflow:hidden; text-overflow:ellipsis; white-space:nowrap; max-width:calc(100%-80px)` |
| P1 | 按钮位置 | 三个模式卡片右侧缺少箭头或操作指示，用户不确定是否可点击 | 添加右侧箭头`>`或点击态背景变化 |
| P1 | 加载状态 | 历史列表无"加载更多"或分页控件，无空状态设计 | 添加"加载更多"按钮或骨架屏 |
| P2 | 时间格式 | "13小时前"/"14小时前"超过24小时仍用相对时间 | 超过24小时显示具体日期 |
| P2 | 空状态 | 未展示无历史记录时的引导页 | 设计空状态引导UI |

## 五、关键发现与优先修复建议

### 必须立即修复 (P0)

1. **聊天页侧边栏内容重复渲染** - body文本中"会话列表"内容出现两次，疑似组件重复挂载Bug
2. **聊天页加号图标重复** - 三个加号图标功能混淆
3. **改图页底部巨大空白** - 有效内容占比仅约15%，首屏利用率低
4. **全景页文件名未截断** - 长文件名会撑破布局

### 建议优先优化 (P1)

5. **首页按钮颜色系统不统一** - 绿/棕/粉三色增加认知负担
6. **改图页上传区域视觉层级弱** - 虚线边框对比度不足
7. **全景页模式卡片缺点击指示** - 用户不确定可点击性
8. **各页面间距系统不统一** - 缺乏8px基线网格
